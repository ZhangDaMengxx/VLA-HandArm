===== 提示词开始 =====

你是一名资深前端 + 设计工程师。请构建一个**单页的机械臂数据回放工作台(Web 应用)**。这是一个日常自用、也用于给客户现场演示的**功能型控制台**,不是营销落地页,不要 hero 大标语、不要"了解更多"这类推销区块。

## 一、这个工作台包裹什么

它是我已有可视化工具 `sim/replay_rerun.py --serve` 的**漂亮外壳**。真正的三联屏渲染(人手视频+骨架 | 机器人 3D | 关节曲线)由 **Rerun 的 web 查看器负责**,我不重建渲染引擎。你要做的是:用一套精致的界面把 Rerun 视口"框"起来,并在外壳里提供换本体、播放控制、A/B 对比、遥测读数这些操作与信息。

**Rerun 嵌入方式**:使用官方 React 组件 `@rerun-io/web-viewer-react` 的 `<WebViewer/>`。

- 数据源是一个 gRPC 代理 URL,形如 `rerun+http://<主机>:9876/proxy`(由 `replay_rerun.py --serve` 启动时打印)。把它作为 `<WebViewer rrd={连接URL} />` 的数据源。
- **关键坑(务必内建为默认行为)**:连接主机要用 **WSL 的 IP**,不能用 `127.0.0.1`(Windows 连不到 WSL 的 127.0.0.1)。所以连接输入框的占位/提示要写明"填 WSL IP,不是 127.0.0.1",并接受粘贴脚本打印的**完整 URL**。
- 用 `hide_welcome_screen` 关掉 Rerun 自带欢迎页;尽量通过 blueprint 把面板锁定为三个视图,让外壳主导观感:①2D 视图(ego 视频 + MediaPipe 骨架)②3D 视图(NERO 臂 + inspire 手装配网格,鼠标轨道旋转)③TimeSeries 视图(关节角曲线)。具体实体路径按 `replay_rerun.py` 里 `rr.log(...)` 用的路径配置,留成可改的常量。
- Rerun 视口默认是**深色**主题,这是**刻意保留的设计**:见下面「设计系统」——外壳纯白极简,视口做成一块深色"仪表屏"嵌进去。

## 二、设计系统(白色极简科技风)

视觉基因取自一个参考页面的**布局语言**(不是它的配色):居中胶囊导航、大面积留白、"浅色外壳里悬浮一块全幅深色内容"、底部悬浮胶囊控制条、克制精致的排版。把它的暖色调**整体翻成白色极简科技风**。

**配色**
- 画布背景 `#F7F8FA`(柔和灰白)
- 卡片/面板表面 `#FFFFFF`
- 仪表屏视口(深色嵌入块)`#0E0F12`(近黑石墨)—— Rerun 视口容器背景
- 主文字 `#16181D`;次要文字 `#6B7280`
- 浅色区分隔线 `#E6E8EC`(1px hairline);深色视口上的分隔线 `rgba(255,255,255,0.08)`
- 主强调色(科技蓝)`#2F6BFF`
- 连接正常/成功 `#12B76A`;警示 `#F79009`;错误 `#F04438`
- A/B 数据色:Raw(原始轨迹)`#9AA3AF` 或 `#F79009`,Stabilized(稳定化后)`#2F6BFF`

**字体**
- 界面与标题:`Inter`(备选 `Geist`),字重 400/500/600,标题字距略收紧
- **所有数值 / 时间码 / 关节角 / 遥测读数用等宽字体**:`Geist Mono`(备选 `JetBrains Mono` / `IBM Plex Mono`)—— 这是"仪表感"的关键
- 允许一个可选的细体衬线只用于极少数强调标签(呼应参考页优雅感),默认可不用

**形状 / 阴影 / 质感**
- 胶囊、按钮:圆角全圆 `9999px`
- 面板、卡片:圆角 `14px`;仪表屏视口 `16px`;输入框 `10px`
- 阴影克制:卡片 `0 1px 2px rgba(16,24,40,.04), 0 8px 24px rgba(16,24,40,.06)`;仪表屏视口阴影略深,读起来像"屏"
- 顶栏与底部悬浮胶囊用**玻璃拟态**:`backdrop-blur(16px)` + 白色 75% 透明 + 1px hairline
- 8px 间距栅格;外边距舒展(24–32px);整体安静、留白多、像精密仪器,不花哨

**动效**:160–220ms ease-out;悬浮胶囊载入时淡入+微上移;hover 仅轻微底色;禁止弹跳/夸张动效。

## 三、页面结构与布局

├──────────────────────────────────────────────────────────────────┤
│   ┌────────────────────────────────────────┐   ┌──────────────┐   │
│   │                                          │   │ 遥测侧栏(可收起) │   │
│   │      ▓▓▓ RERUN 深色仪表屏视口 ▓▓▓          │   │ · 本体 RobotSpec│   │
│   │   三联屏:人手视频+骨架 | 机器人3D | 关节曲线  │   │ · state(13) 读数│   │
│   │   (blueprint 锁三面板,Rerun 自带 chrome 压到最简)│   │ · 管线:规范层→本体层│  │

│            ╭──────────── 悬浮胶囊 · 播放控制 ────────────╮             │
│            │  ▶  ├──●───────────────┤  1.0×  00:03/00:12  ⟨Raw|Stab⟩ │  │

└──────────────────────────────────────────────────────────────────┘

浅色画布四周留白,深色仪表屏视口浮在中间(hairline + 柔和阴影 + 16px 圆角),这是全页视觉重心。

### 3.1 顶栏(约 56px 高,玻璃,通栏)
- 左:字标 `NERO · Inspire`,下方极小副标 `回放工作台`
- 中:**居中胶囊分段导航**——`回放 Replay` / `数据集 Datasets` / `管线 Pipeline`(视图切换,当前选中用白底+主色文字)
- 右:`本体` 下拉(选 RobotSpec,如 NERO)、`任务` 下拉(task 标签)、serve **连接状态圆点**(绿=在线/灰=离线,hover 显示主机与端口)、`设置` 图标

### 3.2 Rerun 仪表屏视口(主区)
- 深色 `#0E0F12` 容器,嵌入 `@rerun-io/web-viewer-react`,填满主区
- 右上角浮一排幽灵按钮:全屏、重连、复制当前连接 URL
- **离线/未连接空状态**(重要,做好看):居中显示引导——先给出可复制命令
  `python sim/replay_rerun.py --serve`
  再给一个输入框粘贴脚本打印的**完整 URL**(占位提示:"用 WSL IP,别用 127.0.0.1;要含 `?url=rerun+http://<WSL-IP>:9876/proxy`"),下方一行小字解释这个坑。连接中显示骨架微光 loading。

### 3.3 遥测侧栏(右侧,约 320px,可折叠成抽屉)
用等宽字体呈现,像仪表盘读数:
- **本体卡**:当前 RobotSpec 名(如 `NERO + inspire`),自由度提示 `7-DoF 臂 + inspire 手`
- **state(13) 实时读数**:分两组——`臂 state[0:7]`(7 个关节角,rad,可配极简 sparkline)、`手 state[7:13]`(6 个驱动)。随时间轴游标更新数值。
- **管线面包屑**:`规范层 canonical_ds` → `本体层 lerobot_ds_NERO`,并标注"规范层本体无关、共享;换本体只重派生本体层"。
- **稳定化参数卡**:`gate_deg`、`oop_alpha` 两个值 + 一行说明"手腕朝向稳定化:抑制出平面漂移、保面内真手势";A/B 模式下附效果注记"臂运动幅度 184°→57°、IK 全收敛"。

### 3.4 悬浮播放胶囊(底部居中,玻璃,参考页 CTA 胶囊的位置与形态)
这是最主要的交互元件:
- 左:圆形 **播放/暂停** 按钮(主色填充)
- 中:**时间轴 scrubber**,带帧刻度;可拖动
- 右:速度切换 `0.5× / 1× / 2×`、时间码 `00:03 / 00:12`(等宽)、**A/B 对比分段开关 `Raw ↔ Stabilized`**
- 播放/暂停/拖动通过 `@rerun-io/web-viewer` 的 API 驱动 Rerun 时间轴(若安装版本支持时间控制);**若该版本不支持程序化时间控制,则保留 Rerun 自带时间轴并压到最简,胶囊改为承载速度、A/B、时间码显示**——请优雅降级,不要假死。

## 四、交互与状态
- **换本体**:下拉切换 → 重新加载对应本体的 Rerun recording(不同 `lerobot_ds_X`/`robot_traj_X`),遥测随之更新;明确提示"规范层不变"。
- **A/B 对比**:切换 = 原始轨迹 `robot_traj_raw` vs 稳定化后 `robot_traj`,用 Rerun 展示两条 recording(切换激活项或并排),遥测同步显示稳定化参数与效果。
- **键盘**:空格 播放/暂停;`←/→` 逐帧;`[` `]` 调速;`F` 全屏。
- **连接状态**:在线绿点/离线灰点;离线时主区走 3.2 的空状态。
- **响应式**:宽度 < 1024px 时遥测侧栏收成抽屉,悬浮胶囊保留,视口继续占主导。
- **可访问性**:键盘可达、焦点环用主色、aria-label 齐全、白底文字对比达 AA。

## 五、领域词典(命名务必用这些,别自造)
- 机器人:**NERO 7-DoF 臂 + inspire 灵巧手**(装配 URDF)
- **规范层 canonical**(本体无关):ego RGB + hand_keypoints(21×3 MANO)+ wrist_pose + task
- **本体层 embodiment**(每机器人一份):state(13) / action(13) / images.ego,其中 **13 = 7 臂关节 + 6 手驱动**
- 换本体 = 加一个 RobotSpec 再重新派生本体层;规范层不动
- 稳定化:`gate_deg`(残差门限剔离群帧)、`oop_alpha`(衰减出平面分量、保面内)
- 源视频 30fps;三联屏共享同一时间轴硬同步
- 可视化数据来自 `robot_traj_*.pkl`(仅回放缓存,非训练数据源)

## 六、技术栈

- React + TypeScript + Vite + Tailwind CSS
- `@rerun-io/web-viewer-react` 作可视化内核
- 字体 Inter + Geist Mono(CDN 或本地);动效可用 framer-motion(可选)
- 组件化拆分(顶栏 / 视口 / 遥测侧栏 / 播放胶囊 / 空状态),清晰可维护
- 交付可直接 `npm install && npm run dev` 跑起来的完整项目

## 七、验收标准
1. 打开即是工作台界面(无营销区);纯白极简科技风,深色 Rerun 视口浮于浅色画布,底部有玻璃播放胶囊,居中胶囊导航齐全。
2. 未连接时展示带命令与 URL 输入、且写明 WSL-IP 坑的漂亮空状态;填入完整代理 URL 后 Rerun 三联屏正常显示。
3. 本体下拉、任务下拉、A/B 分段、速度、时间码、遥测 state(13) 读数全部就位并联动。
4. 播放控制驱动 Rerun 时间轴;不支持时优雅降级并保留 Rerun 原生时间轴。
5. 响应式与键盘可达达标;等宽字体用于所有数值读数。

## 可调项(给你自己,不用粘给 AI)
- **强调色**:现在是科技蓝 `#2F6BFF`。想更"实验室"可换青绿 `#12B76A` 或换成纯中性单色(只留灰阶+一点点主色)。
- **要不要衬线**:极简科技风默认全 sans;若想更贴参考页的编辑感,可让标题用一个细体衬线。
- **侧栏在左还是右**:现写右侧;换左侧也行,顶栏留白相应调整。
- **A/B 呈现**:写的是"切换或并排",若你常做对比,可指定默认并排双视口。
- **Rerun 版本**:程序化时间控制取决于你装的 `@rerun-io/web-viewer` 版本,提示词里已让 AI 优雅降级;若你确定版本支持,可把降级那句删掉逼它必须用 API 驱动胶囊。
- **目标工具差异**:贴给 v0/Lovable 基本可直接出;贴给 Cursor/Claude 建议补一句"先建 Vite 项目脚手架再逐组件实现"。
