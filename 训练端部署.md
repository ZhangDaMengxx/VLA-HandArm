# 训练端部署(Phase C):在 `lerobot_ds` 上从头训 ACT,验证"数据可训"

**目标**:验证我们生成的 LeRobotDataset(`sim/out/lerobot_ds`)能被训练出策略。
先用 **ACT**(小、快、从头训,最直接验证);数据验通后再上 VLA + LoRA(方案 A/B),同一个数据集。

> ⚠️ 这台 CPU-WSL 的 lerobot 0.4.4 只带 VLA(无 ACT/Diffusion),且无 GPU。**训练在 RTX 工作站上做**,按下面执行。

---

## 0. 前提与数据量现实

- 一台带 NVIDIA GPU 的机器。
- 把整个数据集目录拷过去:`sim/out/lerobot_ds/`(含 `meta/ data/ videos/`)。记其路径为 `<DS>`。
- **数据量**:当前只有 **1 episode / 710 帧**。这只够 **冒烟 + 过拟合验证**——目标是"train loss 稳定下降、能过拟合并复现这条 demo",以此证明**数据可被训练管线消费并学习**。要训出有用/可泛化策略,需几十~上百条(靠 B-1 Femto 采集或多视频扩充)。

## 1. 装标准 LeRobot(带 ACT)

在 RTX 上**单独建环境**(别动 CPU 端的 VLA 环境,避免冲突):

```bash
conda create -n lerobot_train python=3.10 -y
conda activate lerobot_train
pip install lerobot          # 标准 LeRobot,ACT/Diffusion 在基础包内
# 或源码:git clone https://github.com/huggingface/lerobot && cd lerobot && pip install -e .
python -c "import pkgutil, lerobot.policies as p; print([m.name for m in pkgutil.iter_modules(p.__path__)])"
# 应能看到 'act'(和 diffusion 等);看不到就换带 ACT 的版本
```

## 2. 离线环境变量(本地数据集必需)

```bash
export HF_HUB_OFFLINE=1
export HF_DATASETS_OFFLINE=1
```
(否则 LeRobot 会去连 huggingface.co 解析版本,离线报网络错——CPU 端已踩过这个坑。)

## 3. 训练 ACT

```bash
lerobot-train \
  --dataset.repo_id=local/nero_inspire_handdemo \
  --dataset.root=<DS> \
  --policy.type=act \
  --policy.device=cuda \
  --batch_size=8 \
  --steps=20000 \
  --save_freq=5000 \
  --output_dir=outputs/train/act_nero
```

- 不同版本 flag 可能略变:先 `lerobot-train --help` 核对;老版本入口是 `python -m lerobot.scripts.train ...` 或 `python lerobot/scripts/train.py ...`。
- ACT 默认用 `observation.state` + `observation.images.ego` → 预测 `action`(动作分块)。我们的数据字段齐全(CPU 端 `sim/check_dataloader.py` 已验 dataloader 可出块状 batch)。
- **成功判据(冒烟)**:train loss 明显下降并能过拟合这 1 条 episode → **"数据可训"验证通过**。

## 4. 评估(两选一或都做)

**A. 行为克隆拟合(动作 MSE)** —— 存成 `eval_bc.py` 在 RTX 跑:
```python
import os; os.environ["HF_HUB_OFFLINE"] = "1"
import torch, numpy as np
from lerobot.datasets.lerobot_dataset import LeRobotDataset
from lerobot.policies.act.modeling_act import ACTPolicy   # 路径按版本核对
DS = "<DS>"; CKPT = "outputs/train/act_nero/checkpoints/last/pretrained_model"
policy = ACTPolicy.from_pretrained(CKPT).to("cuda").eval()
ds = LeRobotDataset("local/nero_inspire_handdemo", root=DS)
errs = []
for i in range(0, len(ds), 20):
    b = ds[i]
    obs = {k: (v.unsqueeze(0).to("cuda") if torch.is_tensor(v) else v) for k, v in b.items()}
    with torch.no_grad():
        a = policy.select_action(obs)
    errs.append(torch.mean((a.cpu() - b["action"]) ** 2).item())
print("action MSE:", float(np.mean(errs)))
```
MSE 很小 = 策略学会复现 demo 动作。

**B. 仿真回放(直观)** —— 用策略逐帧预测 `action`([7臂+6驱动手]),按 `sim/replay_full.py` 的方式喂进 MeshCat 装配回放(把 `robot_traj` 换成策略预测序列),看是否复现 demo。

- **扩数据**:B-1(Femto 采集)或多视频 → 多 episode(单条只验格式)。
