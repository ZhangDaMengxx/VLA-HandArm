# 训练端部署(Phase C):在 lerobot_ds 上从头训 ACT

目的是验证 `sim/out/lerobot_ds` 这套数据能不能训出策略。先用 ACT(小、快、从头训),数据验通了再考虑 VLA 加 LoRA,数据集是同一个。

CPU 端那个 lerobot 只带 VLA,也没 GPU,所以训练放到带 NVIDIA 卡的机器上做。

## 0. 前提

- 一台带 NVIDIA GPU 的机器。
- 把整个 `sim/out/lerobot_ds/`(含 `meta/ data/ videos/`)拷过去,下面记作 `<DS>`。
- 数据量:现在只有 1 个 episode、710 帧,够做冒烟和过拟合验证(loss 稳定下降、能复现这条 demo),说明数据能被训练管线吃进去。要训出有用的策略得几十到上百条,靠 Femto 采集或多段视频补。

## 1. 装 LeRobot(带 ACT)

在 GPU 机器上单独建个环境,别动 CPU 端那个:

```bash
conda create -n lerobot_train python=3.10 -y
conda activate lerobot_train
pip install lerobot
python -c "import pkgutil, lerobot.policies as p; print([m.name for m in pkgutil.iter_modules(p.__path__)])"
```

最后一行应该能看到 `act`;看不到就换个带 ACT 的 lerobot 版本。

## 2. 离线变量

本地数据集加载要设这两个,不然 lerobot 会去连 huggingface.co 报网络错:

```bash
export HF_HUB_OFFLINE=1
export HF_DATASETS_OFFLINE=1
```

## 3. 训练

```bash
lerobot-train \
  --dataset.repo_id=local/nero_inspire_handdemo \
  --dataset.root=<DS> \
  --policy.type=act \
  --policy.device=cuda \
  --batch_size=8 \
  --steps=20000 \
  --output_dir=outputs/train/act_nero
```

不同版本 flag 可能不一样,先 `lerobot-train --help` 对一下;老版本入口是 `python -m lerobot.scripts.train`。ACT 默认用 `observation.state` 加 `observation.images.ego` 预测 `action`,数据集这些字段都齐(`sim/check_dataloader.py` 验过 dataloader 能出块状 batch)。loss 明显下降、能过拟合这一条 episode,就说明数据可训。

## 4. 评估

一是算动作 MSE:加载 checkpoint,在数据集上跑推理,比预测的 `action` 和真值的差,差小就是学会了复现 demo。

二是仿真回放:用策略逐帧预测 `action`([7 臂 + 6 手]),照 `sim/replay_full.py` 的做法喂进 MeshCat 回放,看动作对不对。

## 5. 之后

- 上 VLA:同一个 `lerobot_ds`,`--policy.type` 换成 VLA(lerobot 支持 π0、SmolVLA 等),或回 CPU 端用 xvla 加 LoRA/QLoRA。
- 扩数据:Femto 采集或多段视频,凑够多个 episode。
- 第二个本体(比如 Franka + Allegro)用同一批数据复跑,坐实主体无关性。
