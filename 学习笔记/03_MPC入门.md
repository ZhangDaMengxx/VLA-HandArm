---

## 一句话抓住 MPC

> 别只解"下一步"。把**未来 N 步一起**模拟+优化，找出这一整串怎么走最好，
> 但**只真正执行第 0 步**，下一帧拿新位置**重新解一整张表**。

开车比喻：不是盯着车头前一米，是眼睛望向前方一段路，心里盘算好这段怎么走，
但手上只执行当下这一下方向盘，过会儿再往前望、再算。

---

## 1. 为什么贪心（只解一步）会栽 —— 刹车问题

我原来那套 `solve_qp_step.py` 是**贪心**的：每帧只问
"就现在，末端还差多少，这一步怎么动补最快？" 解完走一步，下帧再问同样的话。
→ 眼里只有当下，**完全不看未来**。

死穴场景 = **刹车**：
车/无人机冲向目标，贪心策略"还远就全油门冲" → 一路冲到跟前 → **刹不住，过冲**。
因为有惯性，不能瞬间停。要想停稳，得在**还没到的时候就提前松油门**。
而"现在该不该刹"取决于**未来会怎样** → 贪心看不到未来的墙 → 每次过冲。

**关键领悟**：有些正确动作现在看是"亏"的（还没到就减速），
只有往前看几步才知道它对。

## 2. 滚动时域（receding horizon）—— 我猜错的地方 ①

我第一反应说 MPC 是"全局规划"。**错了。**

MPC 只往前看**有限的 N 步**（比如未来 20 步 / 2 秒），这段叫**预测时域 horizon**。
看完 → 只走第 0 步 → 整个窗口往前挪一格再看。所以叫**滚动时域**。

为什么不看全局？
- **看不准**：预测靠模型（雅可比线性化），越远越不准。大雾下山，远处算也白算。
- **算不起**：步数越多 QP 越大，实时解不动。机械臂 1000Hz、无人机几百 Hz，每帧都得解完。

→ 务实折中：看一段"看得准又解得动"的距离，走一步，再看。
（上节课"大雾下山"原封不动搬过来，只是现在看一小片坡而不是脚下一步。）

## 3. 决策变量 = 一张 7×N 的表 —— 我猜错的地方 ②③

这里我搅晕过两次，记牢：

**两个方向的"多"，别混：**

- **竖着（7）= 机器人大小**。一帧内 7 个关节各动多少，`Δq` 本身就是 7 个数。这是我早就有的。
- **横着（N）= MPC 新加的**。把这样的列往未来复制 N 份。

```
        第0帧    第1帧    第2帧   ...  第N-1帧
关节1   Δq₀       Δq₁       Δq₂          
关节2    ...
 ...
关节7                                     
       └7个┘                            
       一步QP只解这一列 │ MPC 一口气解整张表 = 7×N 个数
```

**决策变量 = 我能"挑"的、我说了算的东西。**（通勤例子：交通工具能挑，堵不堵车是后果。）

我一开始把这些当决策变量，**全错**：
- 当前 q ❌ —— 已知事实，人就在这，是**起点/输入**，不是待优化。
- 上一步 ❌ —— 过去板上钉钉，没得优化。
- 预测位置 ❌ —— 是我挑完动作后**用模型算出来的后果**（像"会不会堵车"），
  角色是拿去打分（跟目标比），不是决策变量本身。

```
挑一串动作  Δq₀…Δq_{N-1}   ← 决策变量（我说了算）
   ↓ 用模型往前推
一串预测位置 x₁…x_N          ← 后果（算出来的，用来评好坏）
```

→ 决策变量没变出新种类，**就是同一个 Δq 复制 N 份**，被拉长 N 倍。

## 4. "同时解" ≠ "依次解" —— MPC 的命门（我又差点搞混）

我说过一句错的："解一个 QP，再走一遍，填进表里，一直解到位。"
**那是上节课的迭代 IK，不是 MPC。**

- **迭代 IK**：解1列 → 走 → 解1列 → 走 …… 解第0列时**根本不知道**第5列前面有墙。
  还是**贪心**，只是走了很多步 → 照样过冲。
- **MPC**：**一次求解就把整张 7×N 表全部同时解出来**。N 列当成一堆互相牵连的未知数，
  塞进一个大 QP，一起解。

命门在"牵连"：因为一起解，**第 0 列在被解时就"感觉"到了第 N 列前面那堵墙**。
优化器会算"第0列冲太猛 → 第8列刹不住 → 那第0列现在就得收着"。
→ **提前刹车就是这么冒出来的**，依次一列列解永远等不到。

最后一步反直觉：辛苦解出整张 N 列表，**只执行第 0 列，其余全扔**，
下帧拿新位置重解新表。为什么敢扔？越往后越不准（大雾），真正可靠的只有眼前第一步。

```
迭代IK： 解1列→走→解1列→走…      只看当下，贪心，过冲
MPC：   一次解整张N列表→只走第0列→ 扔掉剩下→下帧重解    权衡了未来，提前刹车
```

## 5. 模型必须带质量/速度 —— 我这次答对了

要"预测未来 N 步"，得有个**模型**：现在这状态 + 这动作 → 下帧什么状态。

分水岭：
- **雅可比 = 纯运动学**。（我这次看得准：雅可比其实**已经有速度关系** `ẋ = J·q̇`，
  它天生讲速度的，不是只管位置。）
  但它默认"想多快立刻多快、说停就停" → 没惯性 → 没有过冲这回事 → 用不着 MPC。
- 缺的是"**速度自己怎么变**"。就是**质量/惯性**：
  `F = ma` → 力有限 → 加速度有限 → **速度不能瞬间改，得花时间** → 想停得提前刹。

串起来：
```
有质量 → 速度不能说变就变 → 想停得提前刹 → 必须往前看N步提前算 → 这就是MPC存在的理由
```

带上质量后，状态从只有位置 `q`，升级成**位置+速度 `[q, q̇]`**，
模型从运动学升级成**动力学**。无人机那条线尤其如此（有实打实的质量惯性）。

## 今天立住的 5 块（骨架）

1. 贪心 vs 往前看 —— 只解当下会过冲，得预测未来。
2. 滚动时域 —— 只看有限 N 步（看得准+算得动），走一步再重看。
3. 决策变量 = 7×N 表 —— 未来 N 帧 × 7 关节，一起解。
4. 同时解 ≠ 依次解 —— 一起解，第0步才感觉得到未来的墙 → 提前刹车。
5. 模型必须带质量/速度 —— 纯雅可比没惯性，动力学状态 `[q, q̇]`。
