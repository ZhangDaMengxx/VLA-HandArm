### 3.2 关键项目：统一MPC控制器

**项目目标**：写一个通用的C++ MPC框架，既能控制机械臂又能控制无人机。

```cpp
// 统一MPC接口设计
namespace unified_mpc {

    // 抽象动力学接口
    class DynamicsModel {
    public:
        virtual Eigen::VectorXd f(const Eigen::VectorXd& x, 
                                   const Eigen::VectorXd& u) = 0;
        virtual Eigen::MatrixXd A(const Eigen::VectorXd& x, 
                                   const Eigen::VectorXd& u) = 0;
        virtual Eigen::MatrixXd B(const Eigen::VectorXd& x, 
                                   const Eigen::VectorXd& u) = 0;
    };

    // 机械臂动力学（使用Pinocchio）
    class ArmDynamics : public DynamicsModel {
        // x = [q, q_dot], u = tau
    };

    // 无人机动力学（简化六自由度）
    class UAVDynamics : public DynamicsModel {
        // x = [p, v, q, omega], u = [thrust, tau_x, tau_y, tau_z]
    };

    // 统一MPC求解器（使用CasADi + Ipopt）
    class MPCSolver {
    public:
        MPCSolver(std::shared_ptr<DynamicsModel> model, 
                  int horizon, double dt);
        bool solve(const Eigen::VectorXd& x0,
                   const Eigen::VectorXd& x_ref,
                   std::vector<Eigen::VectorXd>& u_opt);
    private:
        casadi::Opti opti_;
        std::shared_ptr<DynamicsModel> model_;
    };
}
```

**验收标准**：
- 机械臂场景：Panda模型，关节空间轨迹跟踪，满足位置/速度/力矩约束
- 无人机场景：四旋翼模型，位置跟踪，满足推力/姿态角速度约束
- 求解时间：< 50ms（机械臂，N=20）/ < 20ms（无人机，N=20）

### 4.2 统一RL训练框架

**核心思路**：训练一个通用的Residual RL策略网络，通过不同的奖励函数和环境适配，分别服务机械臂和无人机。

```python
class ResidualRLAgent:
    def __init__(self, actor_net, base_planner):
        self.actor = actor_net
        self.base = base_planner

    def plan(self, state, goal, obstacles):
        base_traj = self.base.plan(state, goal, obstacles)
        residual = self.actor(state, goal, obstacles, base_traj)
        opt_traj = base_traj + residual
        return opt_traj

# 机械臂适配
class ArmResidualRL(ResidualRLAgent):
    def reward(self, traj):
        return w1 * clearance_reward + w2 * smoothness_reward + w3 * limit_reward

# 无人机适配  
class UAVResidualRL(ResidualRLAgent):
    def reward(self, traj):
        return w1 * tracking_reward + w2 * avoidance_reward + w3 * energy_reward
```

## 8. 时间分配与精力管理

### 8.1 总体时间预算（假设全职学习/研究）

| 阶段 | 总时间/周 | 共同基础 | 机械臂专精 | 无人机专精 | 通用产出 |
|------|-----------|----------|------------|------------|----------|
| Phase 1（1-3月） | 50h | 40h（80%） | 5h（10%） | 5h（10%） | 0h |
| Phase 2（4-6月） | 50h | 30h（60%） | 10h（20%） | 10h（20%） | 0h |
| Phase 3（7-9月） | 50h | 0h | 25h（50%） | 15h（30%） | 10h（20%） |
| Phase 4（10-12月）| 50h | 0h | 15h（30%） | 15h（30%） | 20h（40%） |

### 8.2 每日节奏建议

```
上午（3h）：共同基础学习/代码
  - 例如：MPC框架开发、CasADi建模

下午（3h）：分轨专精（交替进行）
  - 周一/三/五：机械臂（运动学、路径规划）
  - 周二/四/六：无人机（制导律、多机协同）
  - 周日：通用产出（论文、博客、视频）

晚上（2h）：论文阅读 + 社区参与
  - 阅读顶会论文（RSS/ICRA/IROS/CoRL）
  - 参与ROS/MoveIt/ArduPilot社区讨论
```

### 8.3 精力管理原则

1. **不要并行超过2个深度任务**：上午做共同基础（深度），下午做分轨（一个为主，一个为辅）
2. **每周设定一个"主轨道"**：本周主攻机械臂，下周主攻无人机，交替进行避免疲劳
3. **用产出驱动学习**：每2周必须有一个可展示的产出（代码、文档、视频），避免陷入"一直在学但无成果"
4. **保留缓冲时间**：每月留1周作为缓冲，处理遗留问题或深入某个难点

## 10. 面试时的方向切换话术

### 10.1 应对"你到底想做哪个方向？"

> "我的底层研究兴趣是**智能机器人的统一感知-决策-控制框架**。无论是机械臂还是无人机，核心问题都是相同的：在约束条件下，基于感知信息做出最优决策并生成安全轨迹。我在两个方向都做过深入项目，这反而让我能从交叉视角解决问题。例如，我在无人机RL控制中积累的嵌入式部署经验，可以直接帮助机械臂算法在边缘设备上落地；而我在机械臂运动规划中学习的轨迹优化方法，也可以提升无人机的飞行平滑度。我倾向于选择贵司的具体业务方向，因为我的通用能力可以快速迁移。"

### 10.2 应对"你会不会不够专精？"

> "恰恰相反，我的双轨背景让我对机器人学有了更完整的理解。机械臂让我深入理解了运动学、动力学和轨迹优化；无人机让我深入理解了状态估计、制导律和嵌入式部署。这些知识是互补的，不是互斥的。实际上，顶级机器人公司（如Boston Dynamics、Tesla Optimus）都在寻找既懂机械臂又懂移动机器人的工程师，因为未来的具身智能需要统一的算法框架。"

### 10.3 应对薪资谈判（双轨筹码）

> "我有两个方向的Offer潜力，但我更看重贵司的技术方向匹配度。我的Residual RL框架和视觉EGO方案是跨平台的，无论选择哪个方向都能立即产生价值。如果贵司能提供一个能让我同时接触机械臂和无人机的机会（例如具身智能平台），我愿意在薪资上保持灵活。"
